# 🦁 2주차: 객체지향 I - 클래스와 캡슐화(아기사자 한 명을 하나의 Lion 객체로 표현하는 콘솔 기반 Java 프로그램을 구현해보기)

아기사자 정보를 하나의 객체로 설계하며, **클래스의 역할과 책임, 그리고 캡슐화**를 직접 경험하는 자바 프로그래밍 미션입니다.

## 🎯 과제 목표
* **객체 중심 설계**: 흩어진 변수를 `Lion` 클래스로 구조화하여 객체지향의 기초를 확립합니다.
* **책임의 분리**: 유효성 검증 로직을 `main`에서 `도메인 클래스`로 이동시키며 책임 소재를 학습합니다.
* **캡슐화 경험**: 접근 제어자를 통해 객체의 데이터 보호 원리를 체감합니다.

## 🛠 기술 스택
* **Language**: Java (JDK 17 이상)
* **Environment**: Console (CLI)
* **Key Concepts**: Class/Object, Access Modifiers, Validation

## 📋 주요 기능
1. **Lion 객체화**: 이름, 전공, 기수 데이터를 포함하는 클래스 설계
2. **유효성 검증**: 입력 데이터의 정합성 확인
3. **접근 제어 실험**: `public`, `default`, `private`에 따른 필드 접근 제한 확인

## 🚀 실행 흐름 (Implementation Logic)
본 미션은 객체지향적 설계로 나아가는 과정을 3단계로 구현합니다.

| 단계 | 주요 작업 내용 |
| :--- | :--- |
| **Step 1** | 모든 검증 로직을 `main` 메서드에서 수행 |
| **Step 2** | 검증 로직을 `Lion` 클래스 내부 메서드로 이동 (책임 분리) |
| **Step 3** | 접근 제어자를 적용하여 필드별 접근 범위 테스트 |

## ⚠️ 제약 사항 및 유의 사항
- [ ] **컬렉션 사용 금지**: `List`, `Map` 등을 사용하지 않고 순수 객체 구조에 집중합니다.
- [ ] **출력 로직 분리**: 생성자에는 출력문을 작성하지 않으며, 모든 출력은 `main`에서 담당합니다.
- [ ] **캡슐화 준수**: 접근 제어자 실험 단계에서는 Setter/Getter 없이 필드에 직접 접근하여 확인합니다.
- [ ] **단계별 구현**: Step별 구현 코드를 명확히 구분하여 로직의 이동을 확인합니다.
